Today I'm going to implement a DNS server using `bind9`.
You can check the official documentation here: https://bind9.readthedocs.io/en

The main configuration file is `named.conf`, and it is structured using two concepts: **blocks** and **statements**.

- **Blocks** are containers that group related configuration and define scope.  
  Examples: `options {}`, `logging {}`, `zone example.com {}`.

- **Statements** define specific behaviors or settings inside those blocks.  
  Examples: `directory`, `version`, `recursion`, `severity`, `file`.

For example, the `options` block contains global server settings.

**/etc/bind/named.conf.options** -- file to configure our server
```bash
options {                                  # block options
        directory "/var/cache/bind";       # starting point where bind is going to search
        version "not currently available"; # statement for security

        listen-on {any;};                  # listen in all interfaces for ipv4
        listen-on-v6 {none;};              # we dont use ipv6 so none

        allow-query{any;};                 # to allow receive queries from another pc
        recursion no;                      # to use only as a name server, it will only respond to queries about our zone
};

logging {                                  # block to configure logs

        channel pvazquez_log{             
                file "log/pvazquez.log" versions 3 size 250k;
                severity info;
                print-time yes;
        };

        category default {
                pvazquez_log;
        };

        category queries {
                pvazquez_log;
        };
};
```

**/etc/bind/named.conf.local** -- file to configure zone
```bind
zone "pvazquez.net" {                         # block to configure zone
    type primary;                             # primary zone = master 
    file "/var/cache/bind/db.pvazquez.net";   # zone database file containing all DNS records
};
```

**/var/cache/bind/db.pvazquez.net** -- file to set DNS records
```bind
$TTL 86400
@   IN  SOA alpha.pvazquez.net. admin.pvazquez.net. (   ; alpha is the primary authoritative DNS server for this zone
        2024061308 ; serial
        3600
        1800
        604800
        86400 )

@       IN  NS  alpha.pvazquez.net.                   ;  alpha is the nameserver of pvazquez
alpha   IN  A   192.168.56.103                        ; define the ip of alpha
```

If we want to get more information, we can run: `tail -f /var/cache/bind/log/pvazquez.log`. Then, using nslookup (or dig) from another VM, we should see something like:
`14-Jun-2026 00:47:02.160 client @0x7fb3a8f6e000 192.168.56.1#51602 (pvazquez.net): query: pvazquez.net IN SOA + (192.168.56.103)`

Note: If DNS queries are failing, remember to check your firewall. In my case, nftables was blocking port 53, so I had to allow `tcp dport 53` and `udp dport 53` in the input chain.
