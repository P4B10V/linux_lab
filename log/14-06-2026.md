## Diary

Today I decided to set up a DHCP server. Previously, I had used `isc-dhcp-client` to configure my host‑only interface, so I needed to do some research because I honestly had no idea which package was required for running a DHCP server.

According to what I found, `kea-dhcp-server` is the modern replacement for ISC DHCP, but for this lab I decided to use ISC DHCP even though it is deprecated.  
The documentation I followed is available here: https://kb.isc.org/docs/isc-dhcp-44-manual-pages-dhcpdconf

---

## Network changes

Due to previous configuration steps, I created an Internal Network in VirtualBox called `linuxlab` to replace the old host‑only network.  
This network uses the **10.0.0.0/24** range, with the server assigned **10.0.0.1**.

Because of this change, I had to update:
- the DNS A records to point to the new IP  
- the nftables ruleset to allow DHCP traffic on the internal interface  

Once I confirmed that the new network was working correctly, I proceeded with the DHCP configuration.

---

## DHCP configuration

### File: `/etc/default/isc-dhcp-server`
```bash
INTERFACESv4="enp0s8"
INTERFACESv6=""
```


### File: `/etc/dhcp/dhcpd.conf`
```
ddns-update-style none;
authoritative;

subnet 10.0.0.0 netmask 255.255.255.0 {
    range 10.0.0.200 10.0.0.220;
    option domain-name-servers alpha.pvazquez.net;
    option routers 10.0.0.1;
    default-lease-time 600;
    max-lease-time 7200;
}
```

After applying the configuration, I restarted both bind9 and isc-dhcp-server.

On the admin machine, I released and renewed the IP address:
```
dhclient -r enp0s8
dhclient enp0s8
```

The server logs confirmed that the DHCP process was working correctly:
