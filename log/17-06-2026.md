To explain a bit what Wireguard does, is a VPN where you can easily configure a connection between **peers**. Each of those peers (doesn't matter if it's server/client) has to generate a pair of keys, to do that you can use an example found in `man wg` after you install wireguard.

```
umask 077
wg genkey | tee private.key | wg pubkey > public.key
```

First command (umask) can be ignored, but be aware of the permissions of the files, because it would be the best practice if only root can read/write/execute those. You can do it with `chmod` at configure them, it's your choice when. 

Once we have a public/private key in all of the peers that we want to use VPN, we are going to create de configuration file for each one of them. I used the same path either in the server or client: /etc/wireguard/wg0.conf

**Server** 

This one has been the most difficult one, I had to do a lot of try and error for the postup/postdown rules because I'm not a nft expert. But the other variables you can get it in any page on the internet, even in the man of wg you can check some examples. Even though it was difficult, it was rewarding because when I execute `wg-quick up /etc/wireguard/wg0.conf` it writes the rules to accept VPN and then when I'm done and use `wg-quick down /etc/wireguard/wg0.conf` it just delete the rules from nft, resulting in a firewall who denies everything again, so I'm very happy with the result.

```
[Interface]

Address = 10.100.0.1/24
PrivateKey = **<server_private_key>**
ListenPort = 51820

PostUp = nft add rule inet basic_firewall wg_rules iif "enp0s3" udp dport 51820 accept
PostUp = nft add rule inet basic_firewall wg_rules iifname "wg0" accept

PostDown = nft flush chain inet basic_firewall wg_rules

[Peer]

PublicKey = **<client_windows_public_key>**
AllowedIPs = 10.100.0.100/32

[Peer]

PublicKey = **<client_linux_public_key>**
AllowedIPs = 10.100.0.101/32
```
