**Date:** 15-06-2026

**Log**: Today I was checking my Admin VM. I couldn't figure it out what was happening because ping to 8.8.8.8 was working but i wasn't able to do an `apt update`. First thing It came to my mind it was a DNS error because it couldn't resolve it (I'm using my server as DNS). 

After demolishing my brain I prompt my problem to an AI and the problem as simplier than I thought. BIND's configuration had `recursion: none;` so when I used `apt update` from my AdminVM, my server gets asked about some Debian direction to do apt, but the server response was declined to that. So I had to specify `allow-recursion: 10.0.0.0/24;` and `recursion: yes;` and with that it started to work, BUT it wasn't working. Let me explain.

Even though it was working, I was being deceived because it was my home router who was resolving those queries due to a missconfiguration on my dhcp server with `option domain-name-servers`. I put the domain name there and the correct way was to write `option domain-name-servers 10.0.0.1;`. After restart, I verify that now I'm using my server as resolver.

Moreover, it was necesary to add two ruleset in the nftables output chain:

```
		oif enp0s3 udp dport 53 accept
		oif enp0s3 tcp dport 53 accept
```
