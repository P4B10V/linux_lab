Today I wanted to spend some time learning nftables. In the past I used iptables, and as far as I know, iptables now works on top of nftables, so it makes sense to learn the modern way of managing firewall rules. To start, I decided to read the [official documentation](https://wiki.nftables.org/wiki-nftables/index.php/Main_Page).

The first thing I needed was an SSH rule, since I’m actively working on the server and I don’t want to lock myself out while testing the firewall.

I ran into an issue with my network adapters: each VM has two interfaces (NAT and host‑only), but for some reason enabling one would disable the other. To troubleshoot this, I installed isc-dhcp-client and manually ran dhclient <adapter> (either enp0s3 or enp0s8). This allowed me to bring both interfaces up at the same time.

nftables was installed by default, but the service wasn’t enabled. I checked its status with systemctl status nftables, and since it was disabled and inactive, I enabled and started it with:

    systemctl enable nftables

    systemctl start nftables

Debian ships a default configuration file:
```
#!/usr/sbin/nft -f

flush ruleset

table inet filter {
    chain input {
        type filter hook input priority filter;
    }
    chain forward {
        type filter hook input priority filter;
    }
    chain output {
        type filter hook input priority filter;
    }
}
```

I backed up this file in case I ever need to restore the default configuration, and then I created a separate file to build my own ruleset.

For the input chain, I set the default policy to drop, then added the essential rules:

- iif lo accept — allow loopback traffic
- ct state established,related accept — allow established connections
- tcp dport 9176 accept — allow SSH on my custom port
- ip saddr 192.168.56.0/24 ip protocol icmp accept — allow ICMP only from my local network

I learned that allowing loopback is important because I couldn’t even ping localhost until I added that rule.

This is my final firewall configuration for today:

```
#!/usr/sbin/nft -f

flush ruleset

table inet filter {
    chain input {
        type filter hook input priority filter;
        policy drop;

        iif lo accept
        ct state established,related accept

        ip saddr 192.168.56.0/24 ip protocol icmp accept
        tcp dport 9176 accept
    }

    chain forward {
        type filter hook forward priority filter;
        policy drop;
    }

    chain output {
        type filter hook output priority filter;
        policy accept;
    }
}
```


Some notes for myself:

- iif = input interface
- saddr = source address
- dport = destination port (the port on the server that receives the connection)

I’ll continue exploring nftables in the future, but for now I’m happy with this configuration. It blocks all incoming traffic except loopback, ICMP from my LAN, and SSH on port 9176. This setup is also closer to what I would use on a VPS, where I’d only allow SSH from my public IP.
