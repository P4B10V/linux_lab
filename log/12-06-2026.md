First of all, I had to check on the service, `systemctl status nftables`. It was disable so the next step is obvious, `systemctl start nftables` to run it and `systemctl enable nftables` to make it initiate at boot in case I have to reboot or something. 

Path of interest is `/etc/nftables.conf` with a default configuration:

```
#!/usr/sbin/nft -f

flush ruleset

table inet filter {
    chain input {
        type filter hook input priority filter;
    }
    chain forward {
        type filter hook input priority filter;
    }
    chain output {
        type filter hook input priority filter;
    }
}
```

I'm going to copy that file just in case and use it. 

To start, I had to configure first the necesary ruleset to not get lock out from my SSH session (I'm working through my Admin VM):

- `ct state established,related accept` 
- `tcp dport 9176 accept` -> Current port for SSH


```
#!/usr/sbin/nft -f

flush ruleset

table inet filter {
    chain input {
        type filter hook input priority filter;
        policy drop;

        iif lo accept
        ct state established,related accept

        ip saddr 192.168.56.0/24 ip protocol icmp accept
        tcp dport 9176 accept
    }

    chain forward {
        type filter hook forward priority filter;
        policy drop;
    }

    chain output {
        type filter hook output priority filter;
        policy accept;
    }
}
```
