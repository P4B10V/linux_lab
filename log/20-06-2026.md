After learning a bit about a few services (SSH, nftables, VPN, Samba, DHCP, DNS) now I want to take a step back and create a configuration a bit more professional.

First of all, I'm going to prepare a virtual [router](https://opnsense.org/download/)

It was easy to deploy the VM but the installation was not as intuitive as i though, maybe i didnt read the documentation enough. the first problem that i faced was i couldn't install it on the virtual hard disk, to do that is just simple as log in as the user `installer` and the instalation pops.

Once is complete a reboot it necessary (you have to choose the vhd where you have the configuration).


Right now, my router has two adapters, one as bridge mode and the other as internal network, worth to say that i couldn't access the gui panel through my bridge net, i had to do it with a debian client in the internal network.

If everything was done correctly, visiting https://10.0.0.1 will show the GUI panel of the router (if you read something like "you are running live..." it is necessary to turn off the VM and delete the ISO cd and let only the VHD where you installed OPNsense.


There are a lot of things that you can configure with OPNsense: DNS, DHCP, Firewall, VPN... but I was more interested in the firewall first to see how you can apply rules here.

First, I just wanted to give internet to my client, so I delete all defaults rules and only create one single rule. IN from LAN -> any and with that I could go out to internet, perfect to start the configuration for my server.
Maybe I could stop here and keep practising about OPNsense but right now I have a LAN that is not accesible from outside but from the inside I can reach Internet, also GUI is not accesible from my "WAN".
