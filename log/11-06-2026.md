Today I started my Linux lab by creating two virtual machines running Debian 13: one server in CLI mode and another one to use as an admin workstation.

The first thing I did was configure a host‑only network adapter on both machines so they could communicate directly. After that, I installed openssh-server on the server.

On the admin VM, I generated an SSH key pair using ssh-keygen and copied the public key to the server with ssh-copy-id.

I also applied some basic SSH hardening on the server side:

    - Changed the SSH port
    - Disabled root login (it seems to be disabled by default)
    - Disabled password authentication
    - Allowed only public‑key authentication

I had to be careful while doing this because I didn’t want to lock myself out — even though in this case I could simply open the VM console. Still, I wanted to practice doing it properly, since in the future I might work with a VPS where a mistake like that would be critical.

After finishing the SSH configuration, I wanted to check the logs, but I couldn’t find anything in /var/log. I did some research and learned that Debian 13 uses journalctl for logging instead of traditional log files. Good to know for the next steps.
